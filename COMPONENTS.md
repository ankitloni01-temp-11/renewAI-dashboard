# RenewAI Components Guide

This document explains the internal modules, servers, and technologies running the RenewAI Application. The system is split cleanly between a React-driven Frontend and a highly decoupled Python FastAPI Backend running a multi-agent framework.

## 1. Frontend (React.js + Vite)
Located in `frontend/`, this application surfaces the data generated by the backend components into an interactive Human Investigation Dashboard.
- **Routing:** Handled via React Router for independent sections.
- **Key Dashboards:**
  - **KPIs (`ExecutiveKPIPage.tsx`)**: Global metrics like Conversion rates and Escalations.
  - **Journeys (`JourneysPage.tsx`)**: Node-link mapping of customer journeys across multiple time thresholds natively built.
  - **Prompts (`PromptsPage.tsx`)**: Enables dynamic modification of system and user templates that influence AI execution. 
  - **Agents (`AgentsPage.tsx`)**: Displays throughput metrics (Scans Done, Rejections, etc) for all 10 independent agents.
  - **Audit Log (`AuditLogPage.tsx`)**: Displays the immutable, tamper-proof audit trail for regulatory compliance. 

## 2. Backend Orchestration (FastAPI + LangGraph)
Located in `backend/`, this serves as the API entrypoint and execution hub for customer interactions. 
- **`main.py`**: Mounts all primary web routes to process simulated events and dashboard lookups. 
- **`graph/renewal_graph.py`**: The StateMachine built on **LangGraph**. Maintains the cyclical execution context, allowing agents to propose, critique, and route recursively until achieving safety-approved outcomes. 

## 3. Database Layer (SQLite)
Currently managed via `backend/database/sqlite_manager.py`. It provides storage capabilities for two critical systems:
- **Audit Trails (`audit_log` Table)**: Strict, append-only records of every agent step, the score, compliance verdict, and source tracking. 
- **LLM Prompts (`prompts` Table)**: Real-time dynamic prompt fetching for 14 individual templates affecting Orchestration, Planning, and specific AI Channel executions to remove static hardcoding.

## 4. MCP Servers (Model Context Protocol)
All integrations with databases or external rules run through standard I/O decoupled MCP Server processes. The `MCPClient` handles routing to these servers securely. 
1. **Data Server (`mcp_servers/data_server.py`)**: Covers CRM (Customers, Policies), Journey state reads/updates, Human queue assignments, and basic trigger actions (Emails, SMS, Paylinks). 
2. **Knowledge Server (`mcp_servers/knowledge_server.py`)**: RAG server wrapping **ChromaDB**. Exposes `search_objections` to guide agents in rebuttals and `search_policy_docs` to inject facts.
3. **Safety Server (`mcp_servers/safety_server.py`)**: Enforces the Content Safety Guard. Provides `run_safety_checks` scanning texts for financial mis-selling, PII (Aadhaar/PAN via Regex), or distress keywords.

## 5. Agent Modules
Located in `backend/agents/`, these scripts process LLM completion requests against **Gemini 2.0 Flash** and **Gemini 2.5 Pro**:
- **Execution Agents:** `email_agent.py`, `whatsapp_agent.py`, `voice_agent.py`.
- **Critique Agents:** `email_critique.py`, `whatsapp_critique.py`, `voice_critique.py` (Evaluating channel-specific objectives formatting, conciseness, and tone).
- **Core Agents:** `orchestrator.py` (Determines the sequence of channels), `planner.py` (Determines the message methodology).
- **Human Queue Manager:** `human_queue_manager.py` (Automated handoff of complex scenarios to human operators).
